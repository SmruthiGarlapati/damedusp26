Week 7 — Linked Lists

Prof. Scott · Fall 2024 · Lecture 14 & 15

1. Motivation

Arrays have fixed size and require contiguous memory. Linked lists solve both problems by using dynamically allocated nodes that each hold a pointer to the next.

Key idea: Linked lists trade O(1) random access for O(1) head insertion and flexible size. Know when each matters.

2. Node Structure (C++)

struct Node {
    int val;
    Node* next;
    Node(int v) : val(v), next(nullptr) {}
};

3. Singly Linked List Operations

3.1 Insert at Head — O(1)

void insertFront(Node*& head, int val) {
    Node* n = new Node(val);
    n->next = head;
    head = n;
}

3.2 Insert at Tail — O(n) without tail pointer

void insertBack(Node*& head, int val) {
    Node* n = new Node(val);
    if (!head) { head = n; return; }
    Node* cur = head;
    while (cur->next) cur = cur->next;
    cur->next = n;
}

3.3 Delete a Node — O(n) to find, O(1) to remove

void deleteVal(Node*& head, int val) {
    if (!head) return;
    if (head->val == val) {
        Node* tmp = head;
        head = head->next;
        delete tmp; return;
    }
    Node* cur = head;
    while (cur->next && cur->next->val != val)
        cur = cur->next;
    if (cur->next) {
        Node* tmp = cur->next;
        cur->next = tmp->next;
        delete tmp;
    }
}
head → [1 | ●] → [2 | ●] → [3 | ●] → [4 | nullptr]

4. Doubly Linked List

struct DNode {
    int val;
    DNode* prev;
    DNode* next;
    DNode(int v) : val(v), prev(nullptr), next(nullptr) {}
};

With a direct pointer to a node, deletion becomes O(1) — you can relink prev and next without traversal.

void deleteNode(DNode* n) {
    if (n->prev) n->prev->next = n->next;
    if (n->next) n->next->prev = n->prev;
    delete n;
}

5. Time Complexity Summary

* O(1) if tail pointer maintained separately

6. Common Pitfalls

  • Deleting head before saving head->next → dangling pointer
  • Forgetting to update prev pointer in doubly linked list operations
  • Off-by-one: loop condition cur != nullptr vs cur->next != nullptr
  • Memory leak: always delete removed nodes
Exam tip: Draw the pointer diagram step-by-step. Most lost points are from skipping the diagram and getting the order of pointer updates wrong.
